# 云开发配置指南

## 1. 开通云开发

### 步骤

1. 打开微信开发者工具
2. 点击工具栏「云开发」按钮
3. 首次开通会创建环境，记录环境ID（如：`cloud1-xxx`）
4. 选择「基础版」（免费额度）

---

## 2. 配置环境ID

修改 `miniprogram/app.js`：

```javascript
wx.cloud.init({
  env: 'cloud1-xxx',  // 替换为你的环境ID
  traceUser: true
});
```

修改 `project.config.json`：

```json
{
  "cloudfunctionRoot": "cloudfunctions/",
  "cloudbaseRoot": "cloudbaserc.json",
  "miniprogramRoot": "miniprogram/"
}
```

---

## 3. 创建数据库集合

在云开发控制台 → 数据库，创建以下集合：

| 集合名 | 说明 |
|--------|------|
| users | 用户表 |
| places | 地点表 |
| trips | 行程表 |
| applies | 申请表 |
| wants | 想去表 |
| messages | 消息表 |
| comments | 评论表 |
| notifications | 通知表 |

---

## 4. 导入初始数据

### 方式一：控制台手动添加

1. 打开云开发控制台 → 数据库 → places 集合
2. 点击「添加记录」
3. 复制 `database/init-places.js` 中的数据逐条添加

### 方式二：云函数批量导入

创建临时云函数：

```javascript
// cloudfunctions/init-data/index.js
const cloud = require('wx-server-sdk');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });
const db = cloud.database();

const initialPlaces = [/* 从 database/init-places.js 复制 */];

exports.main = async (event, context) => {
  for (const place of initialPlaces) {
    await db.collection('places').add({ data: place });
  }
  return { success: true, count: initialPlaces.length };
};
```

上传并调用该云函数即可批量导入。

---

## 5. 上传云函数

### 方式一：右键上传

1. 在微信开发者工具中
2. 展开 `cloudfunctions` 文件夹
3. 右键点击 `api` 文件夹
4. 选择「上传并部署：云端安装依赖」

### 方式二：命令行上传

```bash
# 安装云函数依赖
cd cloudfunctions/api
npm install

# 上传（需在开发者工具中）
```

---

## 6. 开启云开发模式

修改 `miniprogram/pages/index/index.js`：

```javascript
data: {
  useCloud: true  // 改为 true
}
```

## 7. 数据库索引配置

在云开发控制台 → 数据库 → 索引管理，添加以下索引：

### users 集合
```json
{"openid": 1}  // 唯一索引
```

### trips 集合
```json
{"placeId": 1}
{"creatorId": 1}
{"date": -1}
{"status": 1}
```

### applies 集合
```json
{"tripId": 1, "status": 1}
{"userId": 1}
{"creatorId": 1}
```

### messages 集合
```json
{"conversationId": 1, "createdAt": -1}
{"toUserId": 1, "read": 1}
```

## 常见问题

### Q: 云函数调用失败

检查：
1. 环境ID是否正确配置
2. 云函数是否已上传
3. 是否有云函数调用权限

### Q: 数据库操作失败

检查：
1. 集合是否已创建
2. 数据库权限配置（开发阶段可设置为「所有用户可读写」）

### Q: 免费额度不够用

建议：
1. 优化查询，减少读操作
2. 使用本地缓存
3. 升级付费套餐

---

## 权限配置

开发阶段建议将数据库权限设置为：

- **读权限**：所有用户可读
- **写权限**：仅创建者可写

生产环境建议使用安全规则：

```json
{
  "read": "auth.openid == doc._openid",
  "write": "auth.openid == doc._openid"
}
```

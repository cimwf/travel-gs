# MVP 第一轮测试用例

目标：用最低成本验证小程序的核心 UI 和行程发布闭环。当前有两类脚本：

- `npm test`：本地页面逻辑自动化 + UI 结构检查，不启动微信开发者工具。
- `npm run test:e2e`：启动微信开发者工具自动化服务，使用 `element.tap()` 对真实小程序页面节点进行点击。
- `npm run test:core`：核心业务流真点击测试，覆盖发起人发布、申请人申请、发起人同意、移除成员、申请人退出、双方行程通知展示。

微信手机号授权、头像选择等系统弹窗仍需体验版或真机人工确认；E2E 脚本会真实点击页面控件，但系统授权返回会用 mock callback 越过。

## 仍需人工真机确认

| 模块 | 原因 | 检查点 |
| --- | --- | --- |
| 微信手机号授权 | 依赖微信客户端系统授权弹窗，本地 Node 无法真实点击 | 首次授权、拒绝授权、再次授权 |
| 头像选择 | 依赖 `chooseAvatar` 系统能力 | 选择头像后预览是否正常 |
| 云数据库真实写入 | 本地脚本只 mock 云函数返回 | 体验版里创建行程后云端 `trips` 是否有记录 |
| 视觉细节 | Node 脚本不渲染真实像素 | iPhone 小屏、常用安卓机、深色模式下是否遮挡或溢出 |
